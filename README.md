### [프로젝트1. DailyRecorder](https://play.google.com/store/apps/details?id=kr.co.yapp.campusrecorder&hl=ko)

### 개요
- 달력 날짜별로 녹음파일 확인이 가능하고, 카테고리 설정을 지원해주는 안드로이드 어플리케이션입니다.


### 프로젝트 명세
* 프로젝트 기간 : 2015.05 ~ 2016.11
* 프로젝트 인원			: 기획자 1명 / 디자이너 1명 / 개발자 2명
* 맡은 역활 (사용 언어) 	: Android 클라이언트 개발자 (JAVA)
* 활용 IDE 				: Eclipse, Android Studio
* 활용 기술 (주요 활용 Framework 및 라이브러리)
 1. 캘린더 구현 <br>(Caldroid Library, Fragment)
 2. 녹음 재생 화면 및 알림 구현<br>(DialogFragment, BroadcastReceiver, Notification)
 3. 카테고리 및 녹음파일 리스트 구현<br>(ListView, ArrayAdapter)
 4. 녹음 파일 관리<br>(SQLITE DB)
* 진행하면서 겪은 어려움
 1. 처음 진행해보는 Android 프로젝트로 인한 어려움 <br>(개발 숙련도와 타 직군간 커뮤니케이션 능력 부족)
 2. 교수님의 사전 허락 없이 진행한 녹음이 불법이라는 사실로 인한 기획 수정 필요
 3. 각자의 일정들로 인한 프로젝트 잠정 중단
* 어려움을 해결한 방법
 1. 스터디 참여 및 개인 공부를 통해 기반 언어(JAVA) 지식을 채우고, 검색을 통해 구현
 2. 타 직군 팀원들과 친해지고 서로 배려하면서 프로젝트를 진행
 3. 예비 사용유저가 아닌 기능성에 초점을 맞춰 재기획

### 프로젝트2. BUU

### 개요
- 가사 분담을 도와주는 일정 관리 서비스

### 프로젝트 명세
* 프로젝트 기간 : 2017.07 ~ 
* 프로젝트 인원			: 기획자 1명 / 디자이너 1명 / 개발자 3명
* 맡은 역활 (사용 언어) 	: Android 클라이언트 메인 개발자, Node.js 서버 공동 개발자
* 활용 IDE 				: Android Studio, Visual Studio Code
* 개발 참여도: **60%**
* 활용 기술 (주요 활용 Framework 및 라이브러리, 사용 기능)
 1. 월별, 주별 캘린더 구현<br>(Caldroid, WeekCalendar Library)
 2. 타임라인 뷰 구현<br>(RecyclerView, ItemTouchHelper)
 3. Server와의 통신 및 push 구현<br>(Retrofit2, FireBase)
 4. 서버 구축 및 DB 모델링 공동 진행<br>(Express Framework, MySQL, MongoDB)
 5. GitHub를 통한 프로젝트 관리<br>(Merge & Pull Request, Commit & Push, MarkDown)
 6. (진행예정) Android MVP 패턴 적용해보기
* 진행하면서 겪고 있는 어려움
 1. 오픈소스 라이브러리 클래스를 상속 받아 일부 커스텀화 하면서 생기는 버그<br>(CustomCalendar.java)
 2. Github, MVP 패턴 등을 적용시켜 보면서 마주치는 예외 상황 및 문제<br>(Git Merge Conflict, MVP 패턴 공부 등)

* 어려움을 해결한 방법
 1. 부족한 개념 및 새로 알게된 개념을 GIT Repository 정리하려 노력함
 2. GitHub Merge를 대비해 오프라인에서 많은 Test 진행 (온라인 진행 시 상황 실시간 공유)
 3. 여러 Test 및 시행착오 과정을 각자 Reference에 관리
 4. 새로운 개념에 대해 각자 개인발표 준비 및 관련 레퍼런스 공유
